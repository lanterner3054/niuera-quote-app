# 更新说明 V3 —— 报价 + PI 双单据系统

更新日期:2026-07-03

## 一、这次更新了什么

1. **新增 PI(Proforma Invoice)生成**:版式 1:1 对齐样例(七列表头、费用行跨列、英文大写金额、红色 Validity、银行信息块、电子章)。
2. **报价一键转 PI**:历史页找到报价 → 点「转为 PI」→ 客户/明细/价格全继承,补个费用行就能出单。可沿用原编号(同号一份报价+一份 PI),也可取新号。
3. **历史存档**:所有生成的单据自动入档,可搜索、重新下载、转单、复制新版。旧文件已全部回填(包括 `D:\报价\PI` 里的历史 PI)。
4. **按报价人分类**:历史页可按报价人筛选。每台电脑会自动记住上次填的报价人姓名。
5. **版本机制**:单据生成后不可改;要改就点「复制新版」,生成 `-R1` 文件,原文件原档不动。
6. **并发安全**:多台电脑同时开单,编号不会重复;同号重复生成会被拦截并提示。

## 二、日常怎么用

- **开报价**:和以前完全一样,「报价」页签,流程无任何变化。
- **开 PI**:
  1. 新客户直接在「PI」页签填写;老单转 PI 走历史页。
  2. 编号默认「生成时自动取号」;转单时可选沿用报价编号。
  3. 「+ 费用行」加运费/清关费等(只填名称、说明、金额)。
  4. 生成前核对表格下方**金色高亮的英文大写金额**。
  5. 银行账户默认中行相城支行,可在设置页增改。
- **历史页**:按类型/报价人/关键字/日期筛选;PI 行显示「源: XNY-ADxxxx」可点击跳转源报价。
- **设置页**:报价人列表(每行一个名字)、银行账户、电子章默认开关。

## 三、需要注意

1. **电子章**:把背景透明的公司章 PNG 放到 `data/stamp.png`(建议 ≥600px 宽)。没放图时 PI 照常生成,只是不带章。
2. **旧档(灰色标签)**:迁移进来的历史 PI/报价 PDF 只能查看和下载原文件,没有明细数据,不能一键转 PI。
3. **阶梯报价转 PI**:数量需要补填,单价按价格档①预填,页面会有黄色提示,务必核对成交价。
4. **条款第 1 条**:新 PI 写的是正确拼写 `Origin`(旧模板拼错为 Orginal,已修正)。
5. xlsx 转 PDF 仍用 WPS/Excel 手动导出(与报价相同)。
